# PRD: Monitoring Layer, Cyclic Arbitrage, and Documentation Updates

## Overview
This PRD outlines the tasks for expanding the `sui-docs` repository with a monitoring layer for liquidations, cyclic arbitrage examples, and updating the token registry and CLI tool documentation.

## Goals
- Implement a monitoring layer to identify liquidatable obligations on Scallop and Suilend.
- Add a cyclic arbitrage example (SUI -> USDC -> CETUS -> SUI).
- Update `manual/14-token-registry.md` with new assets.
- Test and document a set of useful `curl` commands for Sui developers.
- Maintain the "Verification & Build Standard" (type-safety and mainnet compatibility).

## Technical Requirements
- Use Scallop and Suilend SDKs for querying obligations.
- Use Cetus and DeepBook SDKs for arbitrage paths.
- All TypeScript code must be verified with `npx tsc`.
- All code must be compatible with pinned SDKs in `package.json`.

## Task Breakdown

### Phase 1: Documentation & Curl Commands
1. **Verify and Document Curl Commands**: Test all 33 provided curl commands and add them to a new documentation file or existing relevant files (e.g., `manual/utility-tools.md`).
2. **Update Token Registry**: Research recent asset launches on Sui and update `manual/14-token-registry.md`.

### Phase 2: Monitoring Layer
1. **Scallop Obligation Monitoring**: Create a script `scripts/monitor_scallop.ts` that uses the Scallop SDK to query obligations and identify those with Health Factor < 1.
2. **Suilend Obligation Monitoring**: Create a script `scripts/monitor_suilend.ts` that uses the Suilend SDK to query obligations and identify those with Health Factor < 1.
3. **Common Monitoring Utilities**: Extract shared logic (e.g., RPC setup, reporting) into a utility file.

### Phase 3: Cyclic Arbitrage
1. **Arbitrage Strategy Design**: Design the PTB for SUI -> USDC -> CETUS -> SUI.
2. **Implementation**: Create `examples/defi/cyclic_arbitrage.ts` using Cetus and DeepBook SDKs.
3. **Dry-run & Verification**: Verify the arbitrage script against mainnet via dry-run.

### Phase 4: Build Standard & Cleanup
1. **Build Verification**: Ensure all new scripts pass `npx tsc --noEmit`.
2. **Roadmap Update**: Update `README.md` with the completed milestones.
